Esta página te ayudará a instalar y configurar el entorno Haskell para realizar todas las prácticas del **paradigma funcional**.

:::caution

Respetá el orden de instalación de las cosas, algunos componentes necesitan que otros estén previamente instalados.

## GHCup


:::tip

Si tenés una instalación anterior de **Stack, tenés que eliminarla** siguiendo [estos pasos](https://docs.haskellstack.org/en/stable/commands/uninstall_command/).

:::

**GHCup** es una herramienta estándar que facilita enormemente el desarrollo
comercial y académico en Haskell.

Si estás usando MacOS, Linux o WSL, se instala corriendo:

```bash
curl --proto '=https' --tlsv1.2 -sSf https://get-ghcup.haskell.org | BOOTSTRAP_HASKELL_GHC_VERSION=9.10.3 sh

GHCup te va a hacer varias preguntas sobre la instalación, leelas con cuidado
pero las opciones que propone por defecto son apropiadas, podes elegirlas con `Enter`.

## Stack, GHC y HLS

Ahora que tenemos stack instalado podemos instalar las herramientas que vamos
a necesitar para programar que son:

### GHC

Podemos instalarlo ejecutando:

```bash
ghcup install ghc --set 9.10.3
```

[GHC](https://www.haskell.org/ghc/) es nuestro compilador de haskell, que se
encarga de leer nuestro código, y si este satisface algunas condiciones que
ya veremos, darnos un programa listo para ejecutar.

### Stack

Podemos instalarlo ejecutando:

```bash
ghcup install stack --set latest
```

:::note

Es posible que ya este instalado por defecto.

:::

[Stack](https://haskellstack.org/) es el programa con el que vamos a estar
interactuando más directamente y a través de él vamos a compilar y correr
nuestros programas.

### HLS o Haskell Language Server

Podemos instalarlo ejecutando:

```bash
ghcup install hls --set recommended
```

[HLS](https://haskell-language-server.readthedocs.io) es el programa que nos
permite tener una especie de dialogo con el compilador a través de nuestro
editor.

### Chequeo de la instalación de ghcup

Una vez que hayas instalado ghcup, podés comprobar las versiones de las herramientas que instalaste escribiendo en una terminal

```bash
ghcup tui
```

que debería mostrarte una pantalla como la siguiente aunque con diferentes versiones:

![](../../../assets/ghcup-tui-2025.png)

### Otros comandos de GHCup

ghcup permite que vos instales versiones diferentes de GHC, Cabal, stack y HLS. Para eso te recomendamos que utilices la versión gráfica `ghcup tui`. Si sos de utilizar la consola o tenés algún inconveniente con la versión gráfica podés utilizar estos comandos:

```bash
ghcup list  # lista los programas instalados
ghcup help  # ayuda de ghcup
```

## Editor de texto

Nosotros te recomendamos [Visual Studio Code](https://code.visualstudio.com/) pero cualquier editor que entienda lsp va a funcionar.

### Plugins necesarios

Para mejorar nuestra experiencia de Haskell con Visual Studio Code, se deben instalar los siguientes plugins:

- **Haskell**: Haskell language support powered by the Haskell Language Server

Para instalarlo podés ejecutar un comando en vscode (con `Ctrl+Shift+p`):

```
ext install haskell.haskell
```

## Entorno completo

Si todo quedó correctamente instalado, vas a tener un entorno integrado con el código que vayas escribiendo, como te muestra este video

![Linter para Haskell](https://raw.githubusercontent.com/pdep-utn/enunciados-miercoles-noche/refs/heads/master/videos/haskell/vscLinterForHaskell2.gif)

Desplegando el menú `View > Problems (Ctrl + Shift + M)` tenés acceso a todos
los errores y advertencias que tiene tu archivo `.hs`. Al hacer click sobre un
problema te lleva directamente a la línea que origina el inconveniente.

Si tenés algún error, te recomendamos visitar la página [troubleshooting para ayudarte a resolver errores posibles en la instalación de Haskell](/guides/problemas)

:::tip

Es super importante que puedas contar con un entorno integrado, y que no
tengas que escribir el código "a ciegas" para luego compilarlo en una especie
de big bang que te fuerce a mirar todos los errores de una. No es una experiencia
de usuario recomendable, sobre todo en Haskell...

:::
